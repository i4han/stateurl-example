Function transition

  • transition(to, guards?): void

  • Navigate with full guard/transition pipeline

    The transition() function provides the complete navigation experience with:

    • Route guards: Pre-navigation checks with modal/overlay rendering
    • Transition state: Loading indicators and progress tracking
    • Error rollback: Automatic recovery if navigation fails

    Navigation Flow

    1. Parse destination: Prepends basePattern (feature flags) to the path
    2. Auto-lookup guards: Automatically finds guards from route configuration
    3. Set transition state: Updates routerState.transition (triggers loading UI)
    4. Evaluate guards: Checks each guard sequentially
      • If guard blocks: Renders guard component (modal/overlay) with next()/cancel() controls
      • User satisfies condition → calls next() → continues to next guard
      • User cancels → calls cancel() → aborts navigation
    5. All guards pass: Updates browser URL and pathname
    6. Component renders: React renders the new route
    7. Mark stable: Router clears transition state after successful render

    Guard Auto-Lookup

    If no guards are provided, transition() automatically looks them up from the route configuration. This allows you to simply call transition('/path') and the function will find and evaluate any guards defined on that route.

    Parameters

    • to: string

      Destination pathname (absolute path starting with /)

    • Optionalguards: RouteCase[]

      Optional route guards (auto-resolved if not provided)

    Returns void

    Example

    // Simple navigation (auto-resolves guards from route config)
    transition('/dashboard')

    Example

    // With explicit guards
    transition('/admin', [
      { when: () => isAuthenticated, orRender: LoginModal },
      { when: () => isAdmin, orRender: UnauthorizedModal }
    ])

    Example

    // Transition state is available via useTransition() or routerState
    function MyComponent() {
      const transition = routerState.value.transition
      
      return (
        <>
          {transition && (
            <div>Navigating from {transition.from.resource} to {transition.to.resource}</div>
          )}
        </>
      )
    }
