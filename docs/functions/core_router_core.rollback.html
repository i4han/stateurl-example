Function rollback

  • rollback(): void

  • Rollback to last stable location with smooth transition

    This function provides error recovery by reverting to the last known stable page. It's automatically called by the ErrorBoundary when a route render fails, but can also be called manually if needed.

    When it's triggered:

    • Route render error: ErrorBoundary catches the error and calls rollback()
    • Navigation failure: If guards fail or transition is cancelled
    • Manual call: Can be called directly if you detect an invalid state

    Rollback flow:

    1. Capture current location: Take snapshot of where the error occurred
    2. Immediate revert:
      • Update browser URL via replaceState (no history entry)
      • Update pathname signal (triggers re-render)
    3. Set transition state: Show loading bar from error â†’ stable
    4. Visual feedback: Loading bar stays visible for 400ms
    5. Component renders: React renders the last stable page
    6. Clear transition: markStable() clears state after render

    Why it's smooth:

    Instead of an instant hard flash, rollback:

    • Shows the gradient loading bar
    • Provides visual feedback that navigation is occurring
    • Gives user time to understand what happened
    • Creates a polished, professional error experience

    Error page visibility:

    The error page is visible for minimal time (just until React can re-render). The rollback happens immediately, then the loading bar provides smooth visual feedback during the transition back to the stable page.

    Returns void

    Example

    // Automatically called by ErrorBoundary
    <ErrorBoundary onError={rollback}>
      {children}
    </ErrorBoundary>

    Example

    // Manual rollback if you detect invalid state
    if (!user && requiresAuth) {
      rollback()
    }
