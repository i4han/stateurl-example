Function evaluateGuards

  • evaluateGuards(guards): GuardEvaluationResult | null

  • Evaluate route guards sequentially

    Guards are checked in order, stopping at the first one that fails (returns false). This implements the sequential guard pipeline used by the transition() function.

    How it works:

    1. Iterates through guards array in order
    2. Calls each guard's when() function to check the condition
    3. If when() returns false, guard blocks:
      • Returns the guard component and index
      • Caller renders the component as an overlay (modal/dialog)
    4. If all guards pass:
      • Returns null
      • Navigation proceeds to completion

    Sequential vs Parallel

    Guards are evaluated sequentially, not in parallel. This means:

    • Guard 2 only runs if Guard 1 passes
    • Guard 3 only runs if Guards 1 and 2 pass
    • And so on...

    This allows for dependent conditions (e.g., must be logged in before checking subscription).

    Parameters

    • guards: undefined | RouteCase[]

      Array of route guards to evaluate (can be undefined/empty)

    Returns GuardEvaluationResult | null

    First failing guard with its index, or null if all pass

    Example

    // Simple evaluation
    const guards = [
      { when: () => isAuthenticated, orRender: LoginModal },
      { when: () => isPremium, orRender: UpgradeModal }
    ]

    const blocking = evaluateGuards(guards)
    if (blocking) {
      // Guard failed - render blocking.component (LoginModal or UpgradeModal)
      return <blocking.component {...guardContext} />
    }

    Example

    // All guards pass
    const guards = [
      { when: () => true, orRender: SomeModal },
      { when: () => true, orRender: AnotherModal }
    ]

    const blocking = evaluateGuards(guards)
    // blocking === null (navigation proceeds)
