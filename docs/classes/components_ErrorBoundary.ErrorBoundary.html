Class ErrorBoundary

ErrorBoundary - Catch rendering errors and call onError callback

Wraps route content to catch render errors during transitions. When a route fails to render, calls onError() which typically triggers rollback() to revert to lastStable location.

Can be disabled for testing by passing enabled={false}, which allows errors to propagate normally for test assertions.

Example

// Production usage (enabled by default)
<ErrorBoundary onError={rollback}>
  {match ? <RenderRouteMatch match={match} /> : <RenderNotFound />}
</ErrorBoundary>

Example

// Testing usage - disable to let errors propagate
<ErrorBoundary onError={rollback} enabled={false}>
  <ComponentThatThrows />
</ErrorBoundary>
