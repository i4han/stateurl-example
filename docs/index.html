Core Philosophy

URLs are state storage - not just routes. Make URL params feel like regular variables.

AI-native development - AI can autonomously write software that evolves at the speed of thought.

// Instead of navigate('/users/123')
param.userId = '123' // Navigation happens automatically

Design Principles

Seven principles guide StateURL's architecture (see .ai/design-principles.md):

  1. Routing Focus - Excel at URLs, not a framework. No data fetching, forms, or mutations.
  2. Transparency - Direct state access, no hidden internals. All state inspectable and testable.
  3. Single Source of Truth - Route objects referenced, never copied.
  4. Developer Intent - Semantic navigation (via('settings')) over cryptic operators.
  5. Type Safety + Inference + Validation - Types catch errors early, inference speeds up development, runtime validation catches edge cases.
  6. Composition - Provide primitives, let developers compose.
  7. Familiar Terminology - Industry-standard terms (layout, path, breadcrumbs) make intent clear at a glance.
  8. Multi-layered Testing - Unit, component, integration, and E2E tests ensure AI-written code is reliable.
  9. Living Documentation - Documents are AI context, updated in real-time as code evolves.

Key Features

1. Assignable Reactive State

param.userId = '123' // Direct assignment navigates
feature.mode = 'prod' // Features are app-level params

2. Structure-Resilient Navigation

3. Global Accessibility

4. Two-Level State Architecture

  • Features: App-level (/app/:mode/:cycle)
  • Params: Route-scoped (/app/:mode/:cycle/manage/users/:userId)

Core Systems

Signals (Reactivity)

path - Like window.location but reactive (import from stateurl/state/path):

  • path.full - Full pathname (like window.location.pathname)
  • path.base - Base portion extracted from basePattern
  • path.resource - Resource path for route matching
  • path.query - Query string (like window.location.search)
  • path.hash - Hash fragment (like window.location.hash)

Core signals (import from stateurl):

  • pathname - Source of truth signal (low-level, prefer path.full)
  • basePattern - Set by Router component
  • routerState - Contains lastStable, transition, activeGuard for guards/transitions
  • feature / param - Records of individual reactive values

Route Matching

  • matchRouteTree() - Recursive DFS matching
  • getRewrittenSegments() - Index/default rewrites
  • RenderRouteMatch - Component tree rendering

Global Proxies

  • param.* - Global param access
  • feature.* - Global feature access
  • label.* - Label-based route access
  • segment.* - Hierarchical route access
  • router.* - Router state (pathname, basePath, etc.)

Navigation

  • go(url) - Direct navigation
  • via(expr) - Semantic navigation
  • toLabel(name) - Label-based navigation
  • replace(url) - Replace state

Development Patterns

Signal-Based Reactivity

  • All state is signals or computed values
  • Components auto-rerender on signal changes
  • Minimal boilerplate

Proxy-Based API

  • JavaScript Proxies for dynamic access
  • Clean syntax: param.userId not param.get('userId')
  • Future: Full TypeScript inference

Transparent Internals

  • All computed values exposed
  • "Glass box" router for debugging
  • Rich introspection API via RouteAccessor
