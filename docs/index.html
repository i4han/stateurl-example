stateurl 0.3.1 (rhymes with pearl)

state in URL.

React router where URL params and features are assignable reactive variables.

import { feature, useSignals } from 'stateurl'
import type { RouteComponentProps } from 'stateurl'

// url: "/app/dark/products/item/42?sort=price"

function ProductDetail({ param, query }: RouteComponentProps) {
    useSignals()

    // Assign to navigate - URL updates automatically
    param.productId = 43 // ‚Üí /app/dark/products/item/43
    query.sort = 'price' // ‚Üí ?sort=price
    feature.theme = 'dark' // ‚Üí /app/dark/...

    return <div>Product {param.productId}</div>
}

Why stateurl?

The Core Insight

URLs ARE state storage.

But most routers make you jump through hoops to use that storage. stateurl doesn't.

Quick Start

1. Define Routes

import { Router, type Route } from 'stateurl'

const routes = [
    {
        layoutPrefix: '',
        render: Layout,
        outlet: [
            { path: 'home', index: true, render: Home },
            {
                path: 'products',
                render: Products,
                outlet: [{ path: 'item/:productId', render: ProductDetail }],
            },
            {
                path: 'users',
                render: Users,
                outlet: [{ path: 'profile/:userId', render: UserDetail }],
            },
            { path: 'settings', render: Settings },
            { render: MenuNotFound },
        ],
    },
] as const satisfies Route[]

function App() {
    return (
        <Router
            base='app'
            feature={{ theme: ['light', 'dark'] }}
            routes={routes}
        />
    )
}

2. Read & Write State

import { feature, useSignals } from 'stateurl'
import type { RouteComponentProps } from 'stateurl'

function UserDetail({ param }: RouteComponentProps) {
    useSignals() // Enable reactivity

    // Read params from props
    const userId = param.userId // '123'

    // Read/write features (global proxy)
    const theme = feature.theme // 'light'
    const toggleTheme = () => {
        feature.theme = feature.theme === 'light' ? 'dark' : 'light'
        // URL updates: /app/light/... ‚Üí /app/dark/...
    }

    return <div>User: {userId}</div>
}

3. Navigate

import { go, label, handleGo, handleHref } from 'stateurl'
import type { RouteComponentProps } from 'stateurl'
import { products } from './products' // data

// Direct navigation
go('/products/item/1')

// Route accessors (recommended)
label.products.go() // navigate by label
label.userProfile.go() // navigate to labeled route

// Contextual navigation (from component props)
function ItemNav({ param, to }: RouteComponentProps) {
    const nextId = (param.id + 1) % products.length
    const prevId = (param.id + products.length - 1) % products.length

    return (
        <div className='button-group'>
            {/* to() - relative path with interpolation */}
            <button onClick={handleGo(to('$1', [prevId]))}>Prev</button>
            <button onClick={handleGo(to('$1', [nextId]))}>Next</button>

            {/* Anchor with handleHref */}
            <a href={to('/settings')} onClick={handleHref}>
                Settings
            </a>
        </div>
    )
}

üîß Four-Level State Architecture

  • Features: App-level state (theme, mode, locale)
  • Params: Route-scoped state (userId, productId)
  • Query: Component state in query strings (sort, filter)
  • Hash: Scroll position with auto-restoration

All in the URL, all reactive:

/app/dark/users/profile/123?sort=name&filter=active#section
     ^^^^ ^^^^^^^^^^^^^ ^^^  ^^^^^^^^^^^^^^^^^^^^^^^^ ^^^^^^^
 features resource   params  query                    hash

Core Concepts

Routes

Define your app structure:

const routes = [
    // Layout wrapper (no path segment)
    {
        layoutPrefix: '',
        render: Layout,
        outlet: [
            // Basic route
            { path: 'home', render: Home },

            // Index route (matches both /dashboard and as default)
            { path: 'dashboard', index: true, render: Dashboard },

            // With params
            { path: 'users/:userId', render: UserDetail },

            // Nested routes
            {
                path: 'products',
                render: Products,
                outlet: [{ path: 'item/:productId', render: ProductDetail }],
            },
        ],
    },

    // Fallback (no path = catches unmatched)
    { render: NotFound },
]

Base + Feature Props

Extract app-level features from URL:

<Router
    base='app'
    feature={{ theme: ['light', 'dark'], version: ['v1', 'v2'] }}
    routes={routes}
/>

// URL: /app/dark/v2/users/123
// ‚Üí feature.theme = 'dark'
// ‚Üí feature.version = 'v2'
// ‚Üí route path = '/users/123'

RouteComponentProps

Props automatically passed to all route components:

import type { RouteComponentProps } from 'stateurl'
import { handleGo, useSignals } from 'stateurl'

function MyComponent({ param, query, breadcrumbs, to }: RouteComponentProps) {
    useSignals()

    return (
        <div>
            {/* Route params */}
            <p>User: {param.userId}</p>

            {/* Breadcrumbs (matched path segments) */}
            <p>Path: /{breadcrumbs.join('/')}</p>

            {/* Query params - auto-register on assign */}
            <select
                value={query.sort || 'name'}
                onChange={(e) => (query.sort = e.target.value)}
            >
                <option value='name'>Name</option>
                <option value='price'>Price</option>
            </select>

            {/* Navigation */}
            <button onClick={handleGo(to('../'))}>Back</button>
        </div>
    )
}

Route Accessors (Recommended)

Access routes by label or path structure:

import { label, at } from 'stateurl'

// label.* - access by route label (add label prop to routes)
label.admin.go() // navigate to admin route
label.userProfile.go() // navigate to user profile

// at.* - access by path structure
at.users.profile.go() // navigate to /users/profile
at.products.item.param.id // access param

Define labels in routes:

const routes = [
    { path: 'admin', label: 'admin', render: Admin },
    { path: 'users/:userId', label: 'userProfile', render: UserProfile },
]

Query Parameters

Auto-register on assignment - no setup required:

function ProductList({ query }: RouteComponentProps) {
    // Just assign - URL updates automatically
    // /products ‚Üí /products?sort=price&filter=active

    return (
        <div>
            <select
                value={query.sort || 'name'}
                onChange={(e) => (query.sort = e.target.value)}
            >
                <option value='name'>Name</option>
                <option value='price'>Price</option>
            </select>

            <select
                value={query.filter || 'all'}
                onChange={(e) => (query.filter = e.target.value)}
            >
                <option value='all'>All</option>
                <option value='active'>Active</option>
            </select>
        </div>
    )
}

Key Features:

  • Auto-registration: Just assign query.sort = 'price'
  • URL sync: Values appear in query string immediately
  • Shareable: Copy URL to share exact state

Route Guards

Protect routes with conditional rendering:

const authGuard = {
    when: () => !isAuthenticated,
    render: LoginModal,
}

const routes = [
    {
        path: 'admin',
        case: [authGuard],
        render: AdminPanel,
    },
]

// "when not authenticated, render LoginModal"
// User logs in ‚Üí calls next() ‚Üí continues to route
// User cancels ‚Üí calls cancel() ‚Üí stays on current page

Transitions

Track navigation state for loading indicators:

import { useTransition } from 'stateurl'

function Layout({ children }) {
    const transition = useTransition()

    return (
        <div>
            {transition && <LoadingBar />}
            {children}
        </div>
    )
}

API Reference

Core Components:

  • Router - Main router component
  • Outlet - Nested route outlet

Route Accessors:

  • label.* - Access routes by label (label.admin.go())
  • at.* - Access routes by path structure (at.users.profile.go())

Navigation Functions:

  • go(path) - Navigate with pushState
  • replace(path) - Navigate with replaceState
  • handleGo(path) - Curried go() for onClick
  • handleHref - Click handler for anchors

State Access:

  • param - Route parameter proxy
  • feature - Feature flag proxy
  • path - Reactive path object

Hooks:

  • useTransition() - Transition state
  • useSignals() - Enable signal reactivity

Philosophy

URL is State Storage

Most routers: "URL ‚Üí determine UI"

stateurl: "URL ‚Üî state storage ‚Üî UI"

The URL is storage, like localStorage, but shareable:

// Instead of this:
localStorage.setItem('userId', '123')

// Do this:
param.userId = 123 // Shareable, bookmarkable!

Simple is Correct

// This should feel wrong:
navigate(`/users/${userId}/profile`)

// This should feel right:
param.userId = userId
