  • via: ((expression: string, values?: (string | number)[]) => string)

    Resolve via expression to absolute path. Uses 6-level priority search for relative paths. Supports $1, $2... interpolation.

    Example

    via('settings')                    // relative - 6-level priority search
    via('settings/general')            // multi-segment
    via('/admin')                      // absolute (BFS search from root)
    via('user/$1', [userId])           // with $1 interpolation
    via('user/$1/post/$2', [uid, pid]) // multiple placeholders
