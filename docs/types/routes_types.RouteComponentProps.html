  • to: ((path: string, values?: Record<string, string | number | undefined>) => string)

    Resolve path with :param interpolation. Supports relative paths with ../ syntax. Autocomplete available when ToRegistry is augmented.

    Optional Params (:param?)

    Pass undefined to omit the segment - route will use schema default:

    • to('user/:id/:tab?', { id: 1, tab: undefined }) → '/user/1'

    Route Specificity & Order

    Important: The path expression in to() does NOT determine which route you land on. The resulting URL is matched against routes by specificity order.

    When competing routes exist:

    // Routes (correct order: high → low specificity)
    { path: 'item/:sort/:limit', ... }   // matches 2 params
    { path: 'item/:sort/:limit?', ... }  // matches 1-2 params (fallback)

    The to() expression vs actual destination:

    // You write :limit? but provide a value
    to('item/:sort/:limit?', { sort: 'name', limit: 10 })
    // → URL: '/item/name/10'
    // → Lands on: 'item/:sort/:limit' (required route, higher specificity)

    // To land on the optional route, omit or pass undefined
    to('item/:sort/:limit?', { sort: 'name' })
    to('item/:sort/:limit?', { sort: 'name', limit: undefined })
    // → URL: '/item/name'
    // → Lands on: 'item/:sort/:limit?' (optional route)

    Rule: If a competing required-param route exists, pass undefined for optional params to ensure you land on the optional route.

    Example

    to('edit')                                          // relative append
    to('../settings')                                   // go up one level
    to('item/:productId', { productId: 42 })            // :param style
    to('/user/:userId/post/:postId', { userId: 1, postId: 2 })
    to('user/:id/:tab?', { id: 1, tab: undefined })     // omit optional param

  • via: ((expression: string, values?: Record<string, string | number>) => string)

    Resolve via expression to absolute path. Uses 6-level priority search for relative paths. Supports :param interpolation.

    Example

    via('settings')                              // relative - 6-level priority search
    via('settings/general')                      // multi-segment
