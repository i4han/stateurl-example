RouteCase: {

A single route guard condition

RouteCase represents one guard check in the navigation pipeline. When navigation is attempted, guards are evaluated sequentially. If a guard's when condition returns false, navigation is blocked and the orRender component is displayed as an overlay (modal, dialog, etc.) to help the user satisfy the condition.

Sequential Evaluation

Guards are NOT evaluated in parallel. They run one at a time:

  1. Check Guard 1 → if passes, check Guard 2
  2. Check Guard 2 → if passes, check Guard 3
  3. And so on...

This allows for dependent conditions (e.g., must be logged in before checking subscription).

Guard Flow

When a guard blocks:

  1. Navigation is paused (URL doesn't change)
  2. Guard component renders as overlay on current page
  3. User interacts with guard component (login, upgrade, etc.)
  4. Guard component calls next() → continues to next guard
  5. Or calls cancel() → aborts navigation entirely

Type declaration

  • orRender: ComponentType<GuardContext>

    Component to render when guard blocks navigation

    Receives GuardContext props:

    • intended: Where user tried to go
    • current: Where user is now
    • next(): Proceed to next guard
    • cancel(): Abort navigation
  • when: (() => boolean)

    Condition to check before navigation

    Should return true to allow navigation, false to block and render guard. This function is called synchronously during navigation.

      • (): boolean

      • Returns boolean

Example

// Simple authentication guard
{
  when: () => isAuthenticated(),
  orRender: LoginModal
}

Example

// Multiple sequential guards
const premiumRoute = {
  path: 'premium',
  case: [
    { when: () => isAuth(), orRender: LoginModal },      // Check auth first
    { when: () => isPremium(), orRender: UpgradeModal }  // Then check subscription
  ],
  render: PremiumContent
}
